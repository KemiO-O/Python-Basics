## 🚀 Project Overview  

In this project, I learnt how to use Python to:  

👉 Store and work with data

## 🛠️ Skills I Used  

### 🔢 Variables (the building blocks)  
- Stored values like numbers, text, and user input  
- Reused data throughout the program  
- Made the code more flexible and easier to follow  

---

### 🖨️ `print()` and `input()` (talking to the user)  
- Used `print()` to display messages and results  
- Used `input()` to collect user responses  
- Created simple interactive programs  

Basically… turning code into a little chat 😄  


---

### 🔄 Type Casting (sorting out data types)  
- Converted input values into integers or floats  
- Made sure calculations actually worked properly  
- Avoided those confusing “why is this text?!” moments 😅  

---

### 🤔 If Statements (making decisions)  
- Built logic using `if`, `elif`, and `else`  
- Made programs react differently based on user input  
- Created conditions like checking numbers or choices  

---

### 🔁 Loops (doing things more than once)  

**For Loops:**  
- Repeated actions a set number of times  
- Loop through lists or ranges  

**While Loops:**  
- Kept things running until a condition was met  
- Built simple repeating programs (like prompts and menus)

## 📈 What I Built  

Using these skills, I created small programs that:  

- Take user input and respond

- Use conditions to give different outputs  
- Repeat actions until a task is complete  

Simple stuff – but honestly, this is where everything starts ✅

## 🧠 What I Learnt

# 🗄️ SQL Retail & Sales Analysis Project  

Now onto the SQL side of things 👇  

Welcome to my **SQL project** from the same bootcamp – where things get a bit more business-focused 💼  

---

## 🚀 Project Overview  

In this project, I worked with **retail and sales data** to:  

👉 Ask questions using SQL  
👉 Pull the right data  
👉 Turn raw tables into useful insights  

Because let’s be honest… data is only useful if you can actually get answers from it!  

---

## 🛠️ Skills I Used  

### 🔍 SELECT (the starting point)  
- Retrieved specific columns like product names, prices, and sales  
- Built the base for every query  
- Pulled useful datasets from tables  

---

### 🎯 WHERE (finding what matters)  
- Filtered data by product, region, or price  
- Focused on specific conditions (e.g. high sales, certain dates)  
- Made queries more precise  

---

### 📊 ORDER BY (keeping it neat)  
- Sorted results by sales, name, or date  
- Ranked top-performing products or regions  
- Made outputs easier to read  

---

### 🧮 GROUP BY (big picture thinking)  
- Grouped data by category, product, or location  
- Calculated totals, averages, and counts  
- Turned detailed data into clear summaries  

---

### 🔗 JOINs (connecting everything together)  
- Combined tables like customers, orders, and products  
- Matched related data using keys  
- Built complete views of sales activity  

This is where it all *clicks* 👍  

---

## 📈 What I Analysed  

Using SQL, I explored things like:  

- Top-selling products  
- Sales by region or country  
- Customer purchase behaviour  
- Order patterns over time  

Basically answering business questions like:  
👉 What’s selling well?  
👉 Where are we making the most money?  

---

## 🧠 What I Learnt  

- SQL is all about asking the right questions ✅  
- Filtering makes a huge difference  
- GROUP BY helps you see the bigger picture  
- JOINs are where the real magic happens  
- And yes… missing a comma will break everything 😅  

---

## 🌟 Final Thoughts  

This project helped me go from just looking at tables… to actually **understanding the data**  

👉 From Python logic to SQL queries – it’s all about making data useful
